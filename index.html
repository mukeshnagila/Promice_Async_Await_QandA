Q.1 - Write one example explaining how you can write a callback function....?


Ans- A callback is a function passed as an argument to another function This technique allows a function to call another function A callback function can run after another function has finished. The primary purpose of using callback functions is to perform asynchronous operations...

Example-- 
                // function
function start(name, callback) {
    alert('Hi' + ' ' + name);
    callback();
}
// callback function
function callMe() {
    alert('I am callback function');
}
// passing function as an argument
start('Mukesh', callMe);

Explain--
This code defines two functions, 'start' and 'callMe', and demonstrates how to pass a function as an argument to another function.
1- start function takes two parameters, name and 'callback'.
2- It displays an alert saying "Hi" followed by the 'name'.
3- After showing the alert, it calls the 'callback' function.
4- The 'callMe' function is the 'callback' function, which displays an alert saying "I am a 'callback' function".
5- Finally, the start function is invoked with the arguments 'Mukesh' as the name and 'callMe' as the 'callback' function.
When executed, the code will show two 'alerts':
1- "Hi Mukesh"
2- "I am a callback function"

Q.4 - "Create a promise function accepting an argument, if yes is passed to the function then it should go to resolved state and print Promise Resolved, and if nothing is passed then it should go to reject the state and catch the error and print Promise Rejected "....?


Ans- This is the code for this problem and symply you check this in console .... ...

Example-- 
function checkPromise(argument) {
return new Promise((resolve, reject) => {
        if (argument === "yes") {
        resolve("Promise Resolved");
        } else {
    reject(new Error("Promise Rejected"));
        }
    });
    }
    checkPromise("yes")
    .then((result) => {
        console.log(result);
    })
    .catch((error) => {
        console.error(error.message);
    });

Explain--
In this code, the checkPromise function returns a new Promise that accepts an argument. If the argument is equal to "yes", the Promise is resolved with the message "Promise Resolved". Otherwise, it is rejected with an Error object containing the message "Promise Rejected".
In the example usage, we call checkPromise with the argument "yes". Since the argument matches the condition to resolve the Promise, the .then block is executed, and "Promise Resolved" is printed. If you call checkPromise with any other argument (or no argument at all), the Promise will be rejected, and the .catch block will be executed, printing "Promise Rejected" along with the error message.

Explain--

function MainData() {
return new Promise((resolve, reject) => {
        setTimeout(() => {
const data = { name: 'Mukesh', age: 22
        };
        resolve(data);
        }, 1500);
    });
    }
    async function getData() {
    try {
        const data = await MainData();
console.log("return successfully:", data);
    } catch (error) {
console.error("Error find data:", error);
    }
    }
    getData();

Explain--
we define a function "MainData()" that returns a Promise to simulate fetching data from an API. We then create an async function "getData()" that uses async before the function keyword. Inside the "getData()" function, we use await to pause the execution until the Promise returned by "MainData()" resolves or rejects. This makes the code look synchronous and more straightforward to read. In the try block, we assign the resolved data to the data variable and log it. If an error occurs during the async operation, it will be caught in the catch block, and we log an error message.
